# 通用规则（所有引擎共享）

本文件包含引擎无关的图表绘制通用原则。每个引擎的 `_base.md` 处理引擎特定的语法和格式。

## 1. 内容规划

- **一张图一个主题** — 不要把太多信息塞进一张图，宁可拆成多张
- **先理清结构再画图** — 确定节点列表、层级关系、连线关系后再开始生成
- **命名清晰** — 每个节点/元素必须有可读的文字标签

## 2. 布局密度

根据节点数量选择布局密度：

| 节点数 | 密度   | 说明                     |
|--------|--------|-------------------------|
| 1-7    | 宽松   | 大间距、大节点、大字体     |
| 8-15   | 标准   | 中等间距、中等节点         |
| 16+    | 紧凑   | 小间距、小节点、小字体     |

## 3. 中文内容处理

- 中文字符比英文宽，节点宽度至少 ×1.3
- 中文内容的字体大小不应小于 12px
- 标题使用较大字体（18-24px）

## 4. 配色原则

1. **一张图最多用一个配色方案** — 不要混搭
2. **主色不超过 3 种** — 主要 + 次要 + 强调
3. **连线颜色统一** — 除非有语义区分（成功/失败）
4. **分区背景要浅** — 不能抢节点的视觉主体地位
5. **文字颜色比边框深** — 确保可读性

## 5. 质量检查清单

画完图后自查：
- [ ] 所有节点有清晰的文字标签
- [ ] 连线方向一致（不要忽左忽右忽上忽下）
- [ ] 配色不超过 4-5 种主色
- [ ] 没有连线穿过节点
- [ ] 标题存在
- [ ] 内容可读（字体不过小、对比度足够）
